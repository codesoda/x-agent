## Deviations from work plan

- Clean Dockerfile fixtures were simplified (removed `RUN apk add --no-cache curl`) because hadolint DL3018 flags unpinned package versions. Using a minimal `FROM/COPY/CMD` Dockerfile avoids false failures.
- `setup_lock` was intentionally omitted — consistent with gha-agent pattern (single-tool agents don't need lock contention management). The shared lib's lock is optional.

## Edge cases handled
- No Dockerfiles discovered: emits `Result: SKIP (no Dockerfiles found)` and exits 0
- CHANGED_FILES set but no Dockerfile matches: emits `Result: SKIP (no Dockerfiles in CHANGED_FILES)` and exits 0
- build-check defaults OFF (RUN_BUILD_CHECK=0)
- docker not installed when build-check requested: emits `Result: SKIP (docker not found)` and exits 0
- Unknown command: prints usage and exits 2
- help/--help/-h works without hadolint installed
- Per-file hadolint execution with aggregate logging (one bad file doesn't hide others)
- Recursive discovery excludes .git, node_modules, vendor directories
