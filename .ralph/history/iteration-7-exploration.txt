# x-agent Exploration Notes

## Key Files & Roles

- `lib/x-agent-common.sh`: shared boilerplate for x-agent runners. Provides output helpers, tool validation, output directory/cleanup setup, workflow locking, and summary output helpers. Sourced by every runnable agent script.
- `skills/cargo-agent/scripts/cargo-agent.sh`: Rust workflow runner sourcing `lib/x-agent-common.sh`; contains run_x step handlers and domain-specific helpers (jq diagnostics, package resolution, nextest).
- `skills/npm-agent/scripts/npm-agent.sh`: Node workflow runner sourcing `lib/x-agent-common.sh`; contains package-manager detection and file-scoping logic for lint/format.
- `skills/terra-agent/scripts/terra-agent.sh`: Terraform workflow runner sourcing `lib/x-agent-common.sh`; contains safe init/plan behavior and optional tflint checks.
- `skills/bash-agent/scripts/bash-agent.sh`: planned Bash/Shell agent using shared library for syntax checking (`bash -n`) and shellcheck lint.
- `skills/bash-agent/SKILL.md`: planned trigger metadata and allowed tool declarations for shell script checks.
- `tests/run-scenarios.sh`: single orchestrator for shellcheck and fixture execution, including optional filtering by agent/scenario fragment.
- `tests/bash-agent/`: new clean/issues fixture set for FR-13 implementation.
- `tests/*/scenario.env`: declares `SCENARIO_NAME`, `AGENT_SCRIPT`, `RUN_ARGS`, `EXPECT_EXIT`, and `REQUIRED_TOOLS`.
- `tests/cargo-agent/flock/flock-test.sh`: validates lock wait behavior using a competing holder process and waiting-message assertions.
- `install.sh`: installs skills, patches paths, and now includes `lib/x-agent-common.sh` for both local/remote install modes.
- `docs/agents/add-x-agent.md`: canonical structure guide for script boilerplate, helpers, output contract, and agent metadata.
- `docs/agents/scenario-tests.md` and `docs/testing.md`: fixture contracts and local/CI test flow.
- `docs/agents/definition-of-done.md`: acceptance checks for output contract, env knobs, and scenario fixtures.
- `AGENTS.md`: root conventions for Bash 3.2 compatibility, structured output contract, and failure/skip semantics.
- `.github/workflows/ci.yml`: CI entrypoint that runs `tests/run-scenarios.sh` per agent and optional tool-matrix variants.

## Patterns & Conventions

- Script header pattern: `#!/usr/bin/env bash` + `set -euo pipefail`.
- Output format is strict and machine-compatible: each step prints `Step:`, `Result: PASS|FAIL|SKIP`, optional `Fix:`, and `Time:`; run end prints `Overall:` and `Logs:`.
- Runtime knobs are env-overridable: `RUN_<STEP>`, `KEEP_DIR`, `MAX_LINES`, `FAIL_FAST`, `TMPDIR_ROOT`, `CHANGED_FILES`.
- `FAIL_FAST` uses guarded dispatch: evaluate `should_continue()` before each run step when running `all`.
- Temp logs flow through `$OUTDIR`; failures and keep-mode preserve temp directory with `Logs kept in: <path>` from trap; success normally removes temp dir.
- Concurrent invocations are serialized by lockfiles (`${TMPDIR_ROOT}/<agent>.lock`) using `flock` with Perl fallback.
- `CI` detection switches behavior for line truncation and format checks: CI defaults to `MAX_LINES=999999` and check-only modes where applicable.
- Optional tools use skip/fallback behavior rather than hard failure; required tools must trigger `exit 2`.
- Bash compatibility baseline is 3.2: no associative arrays, no `readarray`, no `|&`, no associative case fallthrough.
- New agents should support `--help`/`help`/`-h` before any project-existence checks and reuse `setup_outdir` + `setup_lock` from `lib/x-agent-common.sh`.

## Test Infrastructure

- `tests/run-scenarios.sh` discovers all `tests/*/scenario.env` fixtures, runs `shellcheck` first (or if filtered), and executes each scenario in fixture directories.
- Fixture runner expects relative script path via `AGENT_SCRIPT`; missing script prints `SKIP`.
- If a declared `REQUIRED_TOOLS` item is unavailable, the scenario is `SKIP`.
- `EXPECT_EXIT` values are asserted; mismatch prints log + first 40 lines.
- `tests/cargo-agent/flock` adds explicit lock contention coverage; scenario uses `help` mode to keep run short while validating wait message and eventual completion.
- `tests/run-scenarios.sh --list` enumerates full fixture labels (`tests/<agent>/<case>`), useful to target validation slices.
- CI runs the same command (`tests/run-scenarios.sh <filter>`) and adds optional matrix variants for `nextest` and `tflint`.

## Architecture & Data Flow

- Entry point: one Bash script per agent called from scenario directory context.
- Initialization: set defaults (`KEEP_DIR`, `MAX_LINES`, etc.), validate required tools (`need`), allocate output directory, and obtain lock.
- Execution: parse command args (`--fail-fast`, `help`, and step commands), then route to ordered step functions.
- Each step runs external tool(s), writes full command output to `$OUTDIR/<step>.log`, and prints an abbreviated result with bounded context lines.
- Exit path: steps return shell status to an `overall_ok` accumulator; final block prints `Overall` and returns 0/1.
- Shared-library integration point: each agent script should source `lib/x-agent-common.sh` after `set -euo pipefail` and use `setup_outdir` + `setup_lock` helpers.
