## Deviations from work plan

- **Removed `scoped-clean` scenario**: The test runner (`run-scenarios.sh`) does not support passing arbitrary environment variables (like `CHANGED_FILES`) to agent scripts. The `EXTRA_ENV` feature doesn't exist. Rather than modifying the test runner (out of scope), removed this scenario. CHANGED_FILES scoping is implemented in the script but cannot be exercised via the existing test infrastructure.
- **Item 12 (scope/skip edge cases)**: Implemented `no-chart` scenario for skip behavior. CHANGED_FILES scope testing deferred due to test runner limitation above.

## Edge cases handled
- No `Chart.yaml` found anywhere: returns `SKIP` with explicit reason for both lint and template steps
- `CHART_DIR` set but no Chart.yaml at path: emits reason, SKIP behavior
- `CHANGED_FILES` set but no chart-related files match: emits reason, SKIP
- `CHANGED_FILES` filtering: only `.yaml`/`.yml`/`.tpl` files are considered chart-related
- Chart root deduplication: multiple changed files in same chart produce single invocation
- `--help`/`help`/`-h`: handled before `need helm` so it works without helm installed
- Unknown commands: prints usage and exits 2
- `RUN_LINT=0` / `RUN_TEMPLATE=0` in `all` mode: skips respective steps
- `FAIL_FAST`: uses `should_continue()` from shared lib
- Multiple charts: per-chart sequential iteration with indexed log files
- Subchart exclusion: `find` excludes `*/charts/*` to avoid processing vendored subcharts

## Tests written/run
- `tests/helm-agent/clean/` — valid chart passes with exit 0
- `tests/helm-agent/issues/` — chart with required field error fails with exit 1
- `tests/helm-agent/no-chart/` — empty dir produces SKIP with exit 0
- All 3 scenarios correctly SKIP when `helm` is not installed (REQUIRED_TOOLS gating)
- `shellcheck --severity=warning` passes on helm-agent.sh
- Full suite: all 21 scenarios pass (including all existing agents), 0 failures

## Known limitations
- `CHANGED_FILES` splits on whitespace; paths with spaces are not supported (matches convention used by all other agents)
- Scenario tests require `helm` to be installed to run the actual lint/template checks; they SKIP gracefully when missing
- Test runner does not support EXTRA_ENV, so CHANGED_FILES scoping cannot be exercised in scenario tests
