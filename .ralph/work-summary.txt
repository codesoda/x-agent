## Deviations from work plan

- Simplified playbook discovery: uses recursive `.yml/.yaml` find + `hosts:` grep filter for syntax targets, rather than the more complex filename heuristic patterns (`playbook*.yml`, `site.yml`, `main.yml`) described in the spec. This is simpler and more general — ansible-lint handles its own discovery for linting, and the `hosts:` filter is sufficient for syntax check targeting.
- `setup_lock` was intentionally omitted — consistent with gha-agent/docker-agent pattern (simple agents that don't need lock contention management).

## Edge cases handled
- No YAML files discovered: emits `Result: SKIP (no YAML files found)` and exits 0
- CHANGED_FILES set but no YAML matches: emits `Result: SKIP (no YAML files in CHANGED_FILES)` and exits 0
- No playbooks for syntax check: emits `Result: SKIP (no playbooks found for syntax check)` and exits 0
- FMT_MODE=auto resolves to fix locally, check in CI
- CI forces check mode even when FMT_MODE=fix
- Invalid FMT_MODE exits with code 2
- Unknown command prints usage and exits 2
- help/--help/-h works without ansible-lint/ansible-playbook installed
- Missing required tool exits with code 2
- Per-playbook syntax checking with aggregate failure tracking
- Recursive discovery excludes .git, node_modules, vendor, .venv, .cache, __pycache__

## Tests written/run
- 4 ansible-agent scenarios: all SKIP (ansible-lint not installed locally) — 0 failures
- shellcheck --severity=warning passes on ansible-agent.sh
- Manual verification: --help exits 0 without tools, missing tool exits 2

## Known limitations
- Scenarios require ansible-lint and ansible-playbook installed to run non-SKIP
- Paths with spaces in CHANGED_FILES are not robustly handled (consistent with other agents)
- ansible-lint version differences may produce different rule violations
