# Work Summary — Spec 01: Extract Shared Library

## What was implemented

Created `lib/x-agent-common.sh` containing all shared boilerplate extracted from existing agents:

### Environment defaults
- `KEEP_DIR`, `FAIL_FAST`, `CHANGED_FILES`, `TMPDIR_ROOT` with sensible defaults
- CI-aware `MAX_LINES` (999999 in CI, 40 locally) with explicit override support

### Helper functions
- `hr()` — separator line
- `step(name)` — step header with timing
- `fmt_elapsed()` — elapsed time for current step
- `should_continue()` — fail-fast guard (requires caller `overall_ok`)
- `need(tool)` — required tool check, exits 2 if missing

### Setup helpers (not auto-executed)
- `setup_outdir(agent-name)` — creates temp dir, sets OUTDIR, installs cleanup trap
- `setup_lock(agent-name)` — workflow lock with flock + perl fallback

### Result formatting
- `print_result(ok, log_path, fix_hint)` — prints Result: PASS|FAIL with optional Fix: hint
- `print_overall(overall_ok)` — prints Overall: PASS|FAIL with Logs: path

### install.sh updates
- Remote installs now fetch `lib/x-agent-common.sh` from the repo
- `install_lib_to_root()` function installs lib/ directory (symlink for local, copy for remote)
- Called before skill installation in both Claude and Codex install paths

### run-scenarios.sh updates
- Shellcheck scope expanded to include `lib/` directory

## Deviations from work plan

None. All implementation followed the plan exactly.

## Edge cases handled
- Sourcing produces zero output and no side effects (verified)
- `need()` exits with code 2 for missing tools (verified)
- `setup_outdir` cleanup preserves logs on failure or `KEEP_DIR=1` (verified)
- `setup_outdir` removes temp dir on success with `KEEP_DIR=0` (verified)
- CI MAX_LINES=999999 with `CI=true` and `CI=1` (verified)
- Local MAX_LINES=40 when CI unset (verified)
- Explicit `MAX_LINES` override takes precedence (verified)
- Lock fallback: flock -> perl -> warning (implemented)
- All Bash 3.2 compatible (no associative arrays, no readarray, no `|&`)
