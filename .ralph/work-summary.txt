# Work Summary — Spec 02: Refactor Existing Agents to Use Shared Library

## What was implemented

Refactored all three existing agents (cargo-agent, npm-agent, terra-agent) to source `lib/x-agent-common.sh` instead of duplicating boilerplate.

### For each agent:
1. Added `SCRIPT_DIR`/`LIB_DIR` resolution and `source` line after `set -euo pipefail`
2. Removed duplicated boilerplate: `KEEP_DIR`, `MAX_LINES` (CI-aware block), `FAIL_FAST`, `CHANGED_FILES`, `TMPDIR_ROOT` defaults, `OUTDIR` mktemp, `cleanup()` + trap, `need()`, `hr()`, `should_continue()`, `STEP_START_SECONDS`, `step()`, `fmt_elapsed()`
3. Replaced inline setup with `setup_outdir "<agent-name>"` calls
4. Replaced inline final summary block with `print_overall "$overall_ok"`

### Agent-specific details:
- **cargo-agent**: Also removed inline lock block (flock + perl fallback), replaced with `setup_lock "cargo-agent"`
- **npm-agent**: No lock block existed; kept `detect_pm()`, `_CHANGED_SRC_FILES` filtering, and all `run_script()` helpers. Added `shellcheck disable=SC2034` for `FAIL_FAST` assignment in `--fail-fast` handler (used by `should_continue()` in shared lib)
- **terra-agent**: No lock block existed; kept `normalize_dir()`, `resolve_changed_tf_dir()`, `tf()`, `ensure_terraform_project()`, `normalize_fmt_mode()`. `TF_DATA_DIR_PATH` set after `setup_outdir` to ensure `OUTDIR` is available. Added `shellcheck disable=SC2034` for `FAIL_FAST` assignment

## Deviations from work plan

- npm-agent and terra-agent did not need `setup_lock` — they never had lock blocks. Only cargo-agent had workflow locking.
- Added `shellcheck disable=SC2034` directives for `FAIL_FAST` in npm-agent and terra-agent since shellcheck can't see cross-file usage via sourced library.

## Edge cases handled
- `FAIL_FAST` shellcheck warning: suppressed with SC2034 directive and comment explaining cross-file usage
- terra-agent `TF_DATA_DIR_PATH` depends on `OUTDIR`: placed after `setup_outdir` call
- Agent-specific `RUN_*` variables kept in each agent (not moved to shared lib)
- `SQLX_OFFLINE` export kept in cargo-agent
- `JQ_BIN` kept in cargo-agent
- `detect_pm` and top-level PM echo kept in npm-agent (runs at load time, before main)

## Known limitations
- `print_result` from shared lib is not adopted by any agent in this spec — agents retain their inline Result/Fix printing patterns because each has unique formatting (cargo's JSON diagnostics, terra's init+plan dual logs, etc.). This is intentional to preserve output parity.
- Path coupling (`../../../lib`) remains as designed for this phase.
