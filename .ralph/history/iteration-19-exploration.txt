# x-agent Exploration Notes

## Key Files & Roles

- `lib/x-agent-common.sh`: shared runtime library for all agents. Provides env defaults, step timing, output helpers, cleanup trap, workflow lock, and dependency checking via `need()`.
- `skills/bash-agent/scripts/bash-agent.sh`: multi-scope baseline for `CHANGED_FILES` filtering and output contract.
- `skills/cargo-agent/scripts/cargo-agent.sh`: shared-lib-backed orchestration for heavy multi-step outputs and failure aggregation.
- `skills/go-agent/scripts/go-agent.sh`: domain-specific step toggles and scope mapping for source files.
- `skills/helm-agent/scripts/helm-agent.sh`: Helm chart linter and template validator with auto-detection via Chart.yaml search, CHART_DIR override, and CHANGED_FILES scoping.
- `skills/gha-agent/scripts/gha-agent.sh`: single-step discovery/scoping example, including skip behavior for missing optional directories.
- `skills/npm-agent/scripts/npm-agent.sh`: fallback-tool execution pattern and scoped target filtering for file-based steps.
- `skills/terra-agent/scripts/terra-agent.sh`: lock + non-mutating safety patterns plus optional tooling behavior.
- `tests/run-scenarios.sh`: centralized test runner; invokes shellcheck then executes each `scenario.env` fixture.
- `tests/<agent>/clean/` and `tests/<agent>/issues/`: minimal scenario fixtures for deterministic pass/fail behavior.
- `tests/helm-agent/`: chart fixtures for linting/render validation (clean, issues, no-chart scenarios).
- `install.sh`: installer registry/discovery for skills and optional dependency checks.
- `README.md`: canonical user-facing agent catalog and usage examples.
- `.ralph/specs/06-helm-agent.md`: helm acceptance criteria and required edge-case behavior (`CHART_DIR`, `CHANGED_FILES`, skip semantics).
- `docs/agents/add-x-agent.md`: required files and workflow for adding a new agent.
- `docs/agents/definition-of-done.md`: completion criteria for output contract, knobs, docs, and tests.

## Patterns & Conventions

- Bash scripts are Bash 3.2-compatible and begin with `#!/usr/bin/env bash` plus `set -euo pipefail`.
- Shared helpers are imported via `SCRIPT_DIR/../../../lib/x-agent-common.sh`; no duplicated boilerplate.
- Structured output is mandatory: each step prints `Step`, `Result`, optional `Fix`, `Full log`, and timing; final output includes `Overall` and `Logs`.
- Universal knobs are expected: `RUN_<STEP>`, `KEEP_DIR`, `MAX_LINES`, `FAIL_FAST`, `TMPDIR_ROOT`, and `CHANGED_FILES`.
- CLI contracts require `help`/`--help`/`-h` before mandatory tool checks, `all` default, and explicit command dispatch.
- `RUN_*` and scope checks should be no-op-friendly, preserving deterministic SKIP paths where project content is missing.
- `tests/run-scenarios.sh` resolves paths by directory scan and skips scenarios missing `REQUIRED_TOOLS`.
- `--fail-fast` is enforced with a central `should_continue()` guard around multi-step `all` flows.

## Database Schema

No database is used.

## Test Infrastructure

- Scenario discovery is filesystem-based using `find tests -name scenario.env`.
- Each scenario has a `scenario.env` with `SCENARIO_NAME`, `AGENT_SCRIPT`, `EXPECT_EXIT`, optional `RUN_ARGS`, and `REQUIRED_TOOLS`.
- Scenario tests use clean/issues fixtures with hardcoded fixtures per agent and exit-code assertions.
- `tests/run-scenarios.sh` also runs shellcheck on project scripts, with required severity threshold in CI.
- `.ralph/specs/*.md` encode each spec’s FR checklist; previous iterations highlight recurring gap coverage around scoped-edge cases.

## Architecture & Data Flow

- Invocation enters via `skills/<agent>/scripts/<agent>-agent.sh` and goes through: `help/args -> need checks -> setup_outdir/lock -> discovery/scope -> per-step execution -> final `print_overall`.
- Shared library calls are centralized for lifecycle and formatting; agent scripts contain only domain logic.
- Discovery patterns vary by agent: file extensions, directory conventions, and optional explicit overrides like `CHART_DIR` (helm) or tool-specific root settings.
- Scenario execution flow: discover scenarios, run script from fixture dir, compare observed exit code with `EXPECT_EXIT`, and stream logs on mismatch.
- Security and data-access boundary is local filesystem only; avoid unsafe path expansion and quote discovered paths before execution.
