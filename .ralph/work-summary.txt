# Work Summary — Spec 07: kube-agent (REWORK iteration 3)

## What was implemented

Addressed all remaining review feedback from iteration 26. Two targeted fixes:

### R-008: Network-independent test fixtures
- Added `KUBE_IGNORE_MISSING_SCHEMAS` env knob to `skills/kube-agent/scripts/kube-agent.sh` (default: 0). When set to 1, adds `-ignore-missing-schemas` to kubeconform and `--ignore-missing-schemas` to kubeval.
- Updated ALL kubeconform/kubeval test fixtures (`clean`, `issues`, `scoped-match`, `schema-dir`, `kubeval-clean`, `kubeval-issues`) to `export KUBE_IGNORE_MISSING_SCHEMAS=1`.
- Changed `issues` and `kubeval-issues` fixture manifests to contain YAML parse errors (`containerPort: [invalid`) which kubeconform/kubeval detect regardless of schema availability. This ensures EXPECT_EXIT=1 works even when schemas can't be fetched.
- Updated `usage()` help text in kube-agent.sh to document the new knob.
- Updated SKILL.md allowed-tools and environment knobs table.

### R-011: kubeval wrapper preserves co-located binaries
- Rewrote `tests/kube-agent/run-with-kubeval.sh` to use shadow directories with selective symlinks instead of dropping entire PATH directories. When a PATH directory contains kubeconform, a shadow directory is created with symlinks to ALL binaries except kubeconform. This preserves kubeval (and any other tools) when co-located with kubeconform.
- Shadow directories are created in a temp dir with cleanup trap.
